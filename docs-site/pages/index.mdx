# Trade privately. Clear together.

Quetzal is a MEV-resistant dark-pool DEX on [Aztec Network](https://aztec.network), an L2 ZK-rollup on Ethereum. It is an open-source port of [Penumbra's](https://penumbra.zone) Quetzal protocol — a frequent batch auction with a uniform clearing price — adapted to Aztec's native private state.

Orders are sealed on-chain: side, amount, and limit price are encrypted. The protocol batches every order in a 10-block epoch and computes a single clearing price for all of them at the same time. No order book to front-run. No leaked direction. No MEV bots.

This documentation covers how to use Quetzal on testnet today, how to integrate against the SDK, and how the protocol is built.

<Callout type="warning" emoji="⚠️">
**Testnet alpha.** Quetzal currently runs on the Aztec alpha-testnet (Sepolia L1). Tokens are testnet-only and have no value. An external audit is in progress; do not move real funds yet.
</Callout>

## What makes Quetzal different

- **Sealed orders.** A user's side, amount, and limit price live inside an Aztec note. Only the encrypted commitment is on-chain until the epoch closes.
- **Uniform clearing price.** Every fill in an epoch settles at the same price. No order on the book can be sniped at a worse price than another.
- **K-decoy anonymity set.** Each real order can ship with up to four indistinguishable decoys (Sub-6a) so the order's existence does not pin its sender.
- **Private LP positions.** Liquidity provider positions are notes too. The aggregate pool reserves are public (V3-style buckets); your share in those buckets is not.
- **Permissionless aggregator.** Anyone can register, post a bond, and race to land the next clearing. Treasury pays a per-clearing fee. No protocol-blessed sequencer.
- **Audited L1 bridge.** Canonical USDC, WETH, and wBTC on Sepolia move through a UUPS `TokenBridge` governed by a 3-of-5 multisig + 7-day timelock, plus a 0-delay emergency pauser.

## Status

Quetzal is in alpha on Aztec testnet. The dark-pool clearing path proves end-to-end on testnet; the on-chain recursive-verify clearing tx (`close_epoch_and_clear_verified`) is still being validated against the production aggregator. The L1 bridge is production-grade but operates in a $10k cap mode pending audit. See [Architecture → What's open, what's audited](/architecture#whats-audited) and the [changelog](/architecture#changelog) for the per-sub-project shipping log.

For breaking changes, follow [@github.com/Kubudak90/quetzal](https://github.com/Kubudak90/quetzal).

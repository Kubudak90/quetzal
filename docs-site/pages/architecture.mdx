# Architecture

Quetzal is three tiers running across two chains.

```
┌────────────────────────────────────────────────────────────────┐
│  Browser (Vercel)                                              │
│  ├─ Frontend (React + Vite)                                    │
│  ├─ @quetzal/sdk (Aztec wallet, places orders, claims fills)   │
│  └─ MetaMask (L1 Sepolia, for bridge only)                     │
└────────────────────────────────────────────────────────────────┘
                  │ HTTPS                       │ JSON-RPC
                  ▼                              ▼
┌──────────────────────────┐   ┌────────────────────────────────┐
│  Aggregator (VPS)        │   │  Aztec Network (testnet)        │
│  ├─ Reveal queue          │   │  ├─ Orderbook (Noir contract)   │
│  ├─ Clearing daemon       │──▶│  ├─ Pools 0/1/2 (Noir)          │
│  ├─ Bridge relayer (opt.) │   │  ├─ Treasury                    │
│  └─ Faucet (Next.js API)  │   │  ├─ AggregatorRegistry          │
└──────────────────────────┘   │  └─ Token bridge mode (aUSDC…)   │
                                └────────────────────────────────┘
                                                  │ L1 outbox / L1 inbox
                                                  ▼
                                ┌────────────────────────────────┐
                                │  Ethereum Sepolia (L1)          │
                                │  ├─ TokenBridge.sol × 3         │
                                │  ├─ TimelockController × 2      │
                                │  └─ 3-of-5 multisig             │
                                └────────────────────────────────┘
```

Each tier is described below, then [trust assumptions](#trust-assumptions) at the bottom.

## Contracts on Aztec L2

All Noir contracts. Compiled with `aztec.js` 4.2.1.

### Orderbook

- **`Orderbook.nr`** — sealed-order escrow + epoch state machine + Merkle-rooted fills.
- Storage:
  - `orders: Owned<PrivateSet<OrderNote>>` — per-user sealed orders
  - `current_epoch: PublicMutable<EpochState>` — public epoch counter
  - `fills_root: Map<u32, Field>` — per-epoch settlement Merkle root
  - `treasury: PublicMutable<AztecAddress>` — Sub-5a one-shot setter for the Treasury
  - `pool_registry: PublicMutable<[AztecAddress; 8]>` — Sub-5a mutable pool list (MAX_NUM_POOLS = 8)
- Externals: `submit_order`, `cancel_order`, `close_epoch`, `close_epoch_and_clear_verified`, `claim_fill`
- `close_epoch_and_clear_verified` does a `std::verify_proof_with_type` recursive verify of the clearing-circuit ClientIVC proof + writes `fills_root[epoch_id]` if valid + calls `Treasury.pay_aggregator(winner, fee)`.

### Pools

- **`LiquidityPool.nr`** (3 instances deployed for the testnet triangle).
- Concentrated-liquidity buckets (16 per pool, 1.5× geometric spacing).
- `deposit` / `withdraw` are private (hint-and-validate pattern).
- `apply_clearing(swap)` is called only by the Orderbook contract; iterates a `BucketDelta[4]` sparse encoding.

### Treasury

- Holds tUSDC fee pool. `pay_aggregator(winner, fee)` is `only_orderbook` + silent-partial-pay.

### AggregatorRegistry

- Bond escrow + endpoint discovery.
- Stores `(id, bonded_amount, endpoint_hash, addr)` keyed by `AztecAddress`.
- `view_bonded_amount(addr)` is a `#[view]` so the Orderbook can read it via `self.view(Contract::at(addr).fn())` during a clearing.

### Token (bridge-mode)

- Generic Aztec `Token` contract vendored from `defi-wonderland/aztec-standards@v4.2.0`.
- In bridge mode (`is_bridged: PublicImmutable<bool> = true`) the legacy `mint_to_*` paths revert; only `claim_public`, `claim_private`, `exit_to_l1_public`, `exit_to_l1_private` work.
- In non-bridge mode (used by tUSDC + tETH + tBTC test tokens) `mint_to_private` + `mint_to_public` work as normal; the faucet uses these.

## Aggregator (off-chain)

Lives at `194.163.136.1:3001`. The MVP runs a single instance; the design supports a bonded race of multiple registered aggregators.

- **Reveal queue** — `POST /reveal` accepts a `RevealPayload` (epoch_id, order_nonce, side, amount_in, limit_price, submitted_at_block, owner, submission_tx_hash). Deduplicates by `(epoch_id, order_nonce)`.
- **Validation** — replays the on-chain `order_acc` Poseidon2 hash chain. Tampered reveals are silently dropped.
- **Clearing daemon** — polls `Orderbook.get_epoch()`; when `block >= closes_at_block` + the queue contains matching reveals for that epoch, runs:
  1. `selectBatch` → FIFO 128-order cut
  2. `findClearingPrice` → bisects uniform `P*`
  3. `simulateNet` → routes the imbalance through the pool's buckets
  4. `buildClearingWitnessMultiPair` → witness for the Noir clearing circuit
  5. `runNargoExecute` + `runBbProve` → ClientIVC proof
  6. `submitClearing` → calls `close_epoch_and_clear_verified` on the orderbook
- **Bridge relayer** *(optional, opt-in via `RELAYER_MODE=1`)* — extends the daemon to relay `exit_to_l1` outbox proofs for users who don't want to construct sibling paths themselves; paid via the Treasury fee economy + a per-exit relayer fee.

The aggregator does *not* keep any user secrets. It sees plaintext reveals during clearing (because that's the point of the reveal — to let the clearing circuit work on plaintext inputs), but it doesn't see anything private after the clearing is on chain. The Merkle `fills_root` published on chain is the only way anyone, including the aggregator, learns who got which fill.

See `aggregator/ops/RUNBOOK-aggregator.md` for the production runbook.

## Bridge (L1)

- **`TokenBridge.sol`** (UUPS + AccessControl + Pausable) — one instance per asset (USDC, WETH, wBTC).
- **TimelockController × 2** — one 7-day for `GOVERNANCE_ROLE`, one 0-day for `EMERGENCY_PAUSER_ROLE`. The role admin for the pauser role is *the role itself*, so governance cannot revoke it without acquiring the role + a 7-day window.
- **Multisig × 1** — 3-of-5 Safe (or equivalent) at the top of the governance timelock.
- See [Bridge](/bridge) for the user flow.

## Faucet

- **Next.js Pages API** at `https://faucet.quetzaldex.xyz`
- Endpoints: `POST /api/drip` (CORS + zod + IP-based rate limit + L1 bridge + L2 mint), `GET /api/health`, `GET /api/metrics` (Prometheus exposition format)
- Operator's L1 hot wallet pays Sepolia ETH for the fee-juice bridge; operator's L2 admin wallet mints tUSDC + tETH directly.
- Rate limit: 4 per IP per 8 hours (sliding window).

## Frontend

- **React + Vite** at [quetzaldex.xyz](https://quetzaldex.xyz).
- Screens: Landing (onboarding wizard), Trade, Liquidity, Bridge (deposit/claim/exit tabs), Wallet/History/Settings.
- Uses `@quetzal/sdk` (this repo) + `viem` + `wagmi` (for L1 bridge MetaMask).
- Aztec design tokens: chartreuse (`#D4FF28`) primary accent, ink (`#1A1400`) ground, parchment (`#F2EEE1`) background.

## Trust assumptions <a id="trust-assumptions" />

| Tier | Trust | Mitigation |
|---|---|---|
| Aztec sequencer | Liveness only (no value at risk) | Permissionless clearing-aggregator race; users can self-bond + clear if no one else does |
| Clearing aggregator (off-chain) | Reveals plaintext orders to whichever aggregator wins the race | Sub-6d threshold T-of-N or rotating-aggregator commitments (future) |
| Aztec Labs L1 RPC | Liveness; throttling currently blocks Sub-9.4 end-to-end | Self-host Aztec node with non-Quicknode L1 RPC (future) |
| L1 bridge multisig | 3-of-5 governance multisig can upgrade `TokenBridge.sol` (UUPS) | 7-day timelock + on-chain visibility |
| L1 bridge emergency pauser | Separate multisig with 0-day timelock for `pause()` only | Self-admin invariant: governance cannot revoke pauser without first acquiring the role |
| Faucet operator | Sees IP + L2 address mapping (rate limit + audit log) | No personal data collected beyond IP; logs rotate per Sub-7a runbook |
| Frontend (Vercel) | Static React bundle; XSS would let an attacker rewrite calldata | Subresource integrity not yet enforced; CSP is permissive; **don't trade real value on testnet** |

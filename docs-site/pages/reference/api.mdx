# HTTP APIs

Quetzal exposes two minimal HTTP services. Both run on the operator's VPS / Vercel infra; neither holds user secrets.

- **Aggregator** — reveal ingestion + clearing daemon (port 3001 on `194.163.136.1`)
- **Faucet** — Aztec drip pipeline (`https://faucet.quetzaldex.xyz`)

## Aggregator API

Base URL: `http://194.163.136.1:3001` (HTTP only until DNS for `aggregator.quetzaldex.xyz` is wired + Caddy issues TLS).

### `POST /reveal`

Submits a plaintext reveal of an order placed in the previous or current epoch. The aggregator buffers reveals in memory until the epoch closes, replays the `order_acc` Poseidon2 chain against on-chain state, and includes valid reveals in the next clearing.

**Request body** *(zod-validated)*:

```ts
{
  epoch_id: number;          // u32, the epoch the order was placed in
  order_nonce: string;       // 0x-prefixed Field
  side: boolean;             // false = buy canonical-low, true = sell
  amount_in: string;         // decimal bigint string
  limit_price: string;       // decimal bigint string (18-dec fixed)
  submitted_at_block: number; // u32, L2 block number of submit
  owner: string;             // 0x-prefixed AztecAddress
  submission_tx_hash?: string;
}
```

**Response**:

```json
{ "ok": true }
```

**Errors**:

- `400 { "error": "invalid payload", "issues": [...] }` — zod validation failed
- The aggregator does *not* reject duplicates at HTTP time; dedup happens in the queue layer keyed by `(epoch_id, order_nonce)`.

```bash
curl -X POST http://194.163.136.1:3001/reveal \
  -H "Content-Type: application/json" \
  -d '{
    "epoch_id": 47,
    "order_nonce": "0xabc...",
    "side": false,
    "amount_in": "1000000",
    "limit_price": "1500000000000000000",
    "submitted_at_block": 12345,
    "owner": "0x..."
  }'
```

In practice, you don't call this directly. The SDK's `client.aggregator.broadcastReveal()` and `client.aggregator.directReveal()` wrap it.

### `GET /health`

Liveness + watcher state.

**Response**:

```json
{
  "ok": true,
  "service": "quetzal-aggregator",
  "queueSize": 0,
  "watcher": {
    "status": "polling",
    "lastEpochSeen": 47,
    "lastBlockSeen": 12345,
    "lastError": null,
    "lastPollAt": "2026-05-29T17:00:00.000Z"
  }
}
```

`watcher.status` values:

| Status | Meaning |
|---|---|
| `polling` | Healthy; watching the orderbook |
| `closing` | Currently executing a clearing tx |
| `error` | Last poll or clearing failed; see `lastError` |

The frontend's landing-page status pill (`live` / `degraded` / `down`) probes this endpoint + the faucet `/api/health` and ORs the results.

## Faucet API

Base URL: `https://faucet.quetzaldex.xyz`.

### `POST /api/drip`

Drips fee-juice + tUSDC + tETH to an Aztec L2 address. See [Get test tokens](/getting-started/faucet) for the user flow.

**Request body**:

```ts
{
  address: string;          // 0x-prefixed AztecAddress
  captchaToken?: string;    // hCaptcha response token (optional during bypass)
}
```

**Response (200)**:

```json
{
  "success": true,
  "l1TxHash": "0x...",
  "l2ClaimAmount": "100000000000",
  "l2LeafIndex": 95092738,
  "tUSDCTxHash": "0x...",
  "tETHTxHash": "0x..."
}
```

**Errors**:

- `400 { "success": false, "error": "invalid body: ..." }` — zod validation failed
- `403` — CORS Origin not in allowlist
- `429 { "success": false, "error": "rate limit exceeded" }` — IP exceeded 4 drips per 8 hours
- `503 { "success": false, "error": "drained" }` — operator's L2 mint balance below threshold

CORS: only the configured allowlist (frontend prod + preview Vercel URLs) is accepted. Cross-origin requests from elsewhere get `403`.

A successful call takes 2–4 minutes wall time (ClientIVC proof gen + L1 inbox finalisation + L2 mint).

<Callout type="warning">
  Don't script bulk drips. The faucet operator pays real Sepolia ETH for every L1 deposit; abuse depletes the hot wallet and degrades testnet for everyone.
</Callout>

### `GET /api/health`

Same shape as documented in [Get test tokens → Health endpoint](/getting-started/faucet#health-endpoint).

### `GET /api/metrics`

Prometheus exposition-format metrics. Used by the Sub-5c monitoring stack. Sample exported keys:

- `faucet_l1_balance_eth`
- `faucet_l1_balance_feejuice`
- `faucet_l2_balance_tusdc`
- `faucet_l2_balance_teth`
- `faucet_drip_total{result="ok|err"}`
- `faucet_drip_duration_seconds`
- `faucet_rate_limit_unique_ips`
- `faucet_rate_limit_windowed`

Scrape interval is operator-configured; the public Prometheus is operator-only.

## Authentication

Neither API requires authentication for the public endpoints. Rate limiting is by IP + CORS Origin only.

For aggregator registration (which is a privileged on-chain operation, not an HTTP one), see `client.aggregator.register()` in the [SDK reference](/reference/sdk#aggregator-api).
